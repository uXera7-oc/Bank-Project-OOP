# C++ OOP Bank Management System

This repository contains a **console-based Bank Management System** developed using **C++ and Object-Oriented Programming (OOP)** principles.

The project simulates a simplified **banking environment** where users interact with the system according to their assigned permissions while performing core banking operations.

The main objective of this project is to apply **real-world software design concepts** such as **Divide and Conquer**, **reusability**, and **clean code architecture**, while strengthening practical understanding of **OOP principles**.

📹 The repository also includes a **demo video** demonstrating the system and its main features.

---

## 📂 Project Overview

The system provides a structured banking simulation with several core features, including:

- 👤 User authentication  
- 🔐 **Permissions-based access control**  
- 👥 Client management  
- 💰 Banking transactions  
- 📊 Transaction monitoring  
- 📜 **Logs tracking system** for recording system activities  
- 🖥️ Console-based interface  

The application is designed with **clear separation of responsibilities** to maintain organized and readable code.

---

## 🧱 Core OOP Concepts Covered

### 🔹 Encapsulation
Sensitive data is protected through controlled access mechanisms, ensuring that internal data remains secure and consistent.

---

### 🔹 Abstraction
Complex operations are simplified into clear system interfaces, allowing users to interact with the system without needing to understand internal implementation details.

---

### 🔹 Inheritance
Shared behaviors and structures are reused across different parts of the system to reduce duplication and improve maintainability.

---

### 🔹 Polymorphism
Flexible method behavior allows the system to handle different operations through unified interfaces.

---

## 🧠 Software Design Principles Used

### Divide and Conquer
The system is organized into independent components, where each part handles a specific responsibility, making the project easier to understand and maintain.

---

### Reusability
Reusable structures and logic are implemented throughout the system to minimize repetition and improve efficiency.

---

### Code Readability
Special attention is given to writing **clean, structured, and readable code**, making the project easier to follow and maintain.
